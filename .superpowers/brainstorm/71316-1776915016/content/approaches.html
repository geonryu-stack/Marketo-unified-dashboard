3가지 설계 방향

새 발송 설정 페이지의 구조를 어떻게 잡을지 — 하나를 골라주세요

① 에셋 선택 + URL 입력
② [테스트 발송] 버튼 → 확인
③ 그룹별 날짜·시간·RTZ/KST 설정
④ 전체 예약 완료

A. 단계별 위저드

에셋 → 테스트 → 그룹 설정 → 예약 순서로 진행. 순서가 명확하고 실수 방지에 좋음.

장점

  • 흐름이 명확
  • 테스트 완료 확인 후에만 다음 단계 진행

단점

  • 클릭 수 많음
  • 전체 그림이 한눈에 안 보임

B. 올인원 단일 페이지 ⭐ 추천

에셋+URL이 상단에, 4개 그룹 카드가 하단에 한눈에 보임. 1주일치 스케줄을 한 화면에서 설정.

장점

  • 전체 그림이 한눈에
  • 빠른 설정 (스크롤 없이)
  • 주간 스케줄링에 최적

단점

  • 정보 밀도가 높아 초기에 복잡해 보일 수 있음
캠페인 히스토리 목록

C. 히스토리 목록 + 사이드 드로어

메인 화면에 발송 이력이 항상 보이고, 우측에서 드로어가 열려 새 발송 설정.

장점

  • 이전 발송 참조하며 설정 가능

단점

  • 그룹 카드 공간 협소
  • 구현 복잡도 높음
