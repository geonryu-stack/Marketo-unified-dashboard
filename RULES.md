# Marketo Send Automation — 제약사항 및 운영 규칙

> 이 문서는 발송 자동화 프로젝트의 모든 현실적 제약사항을 기록한 통합 문서입니다.

## 프로젝트 개요

Marketo를 통해 프로모션 이메일을 발송하되, 발송 대상자 데이터는 사내 DB에서 추출하는 워크플로우를 자동화하는 웹 애플리케이션.

**핵심 목표**:
1. 사내 DB 연동 → 세그먼트(대상자) 추출
2. 추출된 대상자 → Marketo 특정 리스트에 업로드
3. 에셋(이메일 내용) 선택 → 보상 URL 삽입 → 예약 발송

---

## 현실적 제약사항

### [CONSTRAINT-01] 사내 DB 읽기 전용 접근
- 사내 DB는 **백업 DB로만 활용** 가능 (READ ONLY)
- 어떠한 INSERT / UPDATE / DELETE 쿼리도 사내 DB에 직접 실행 불가
- 세그먼트 추출 쿼리는 반드시 SELECT 문만 사용
- 추출된 데이터는 앱 내부 SQLite DB에 임시 저장 후 처리

### [CONSTRAINT-02] 보상 URL 수동 발행
- 이메일에 삽입되는 인앱 보상 URL은 **반드시 사람이 수동으로 발행**해야 함
- 시스템이 자동으로 URL을 생성하거나 변조할 수 없음
- 워크플로우: 담당자가 URL 발행 → 웹 UI의 지정 입력 필드에 붙여넣기 → 시스템이 템플릿 URL 위치에 자동 치환

### [CONSTRAINT-03] Marketo 에셋 관리 방식
- 매 발송 시 Marketo 내에서 **지난 회차의 에셋을 Clone**하여 진행
- Clone된 에셋의 특정 URL 위치에 새 보상 URL을 대체하는 방식
- 또는 새로 생성 시: 이미지 에셋에 맞는 title, emoji, preheader, 내부 텍스트를 모두 새 내용으로 교체

### [CONSTRAINT-04] 에셋 다양성 요구
- 매 발송마다 **다른 이미지, 타이틀, 내부 템플릿 문구** 사용
- 이미지별로 연관된 메타데이터 세트(title, emoji, preheader, body text) 미리 준비 필요
- 에셋 라이브러리를 사전에 충분히 쌓아두는 전략 필요

### [CONSTRAINT-05] Marketo 자체 DB로는 자동화 불가
- Marketo의 자체 저장 DB는 사내 DB와 동기화되지 않음
- 발송 대상자 추출은 반드시 사내 DB 기반으로 수행
- Marketo는 발송 엔진으로만 활용 (리스트 업로드 → 캠페인 실행)

## 워크플로우 설계

```
[사내 DB] ──READ ONLY──▶ [세그먼트 빌더] ──▶ [대상자 CSV/리스트]
                                                      │
                              [에셋 라이브러리]        │
                              (이미지+텍스트 세트)      │
                                    │                 │
                                    ▼                 ▼
                         [캠페인 생성 위저드]  ◀────────┘
                                    │
                                    ├── 보상 URL 수동 입력
                                    ├── 에셋 선택
                                    ├── Marketo 리스트 선택/생성
                                    └── 발송 예약 설정
                                             │
                                             ▼
                              [Marketo API] ──▶ [예약 발송]
                                             │
                                             ▼
                               [캠페인 대시보드 (상태 모니터링)]
```

---

## 기술 제약

- 내부 DB 드라이버: MySQL2 (읽기 전용 연결)
- 앱 자체 DB: SQLite (에셋, 세그먼트 쿼리, 캠페인 이력 저장)
- Marketo 연동: Marketo REST API (OAuth 2.0 Client Credentials)
- 환경변수로 DB 접속 정보 및 Marketo API 키 관리 (.env.local)
